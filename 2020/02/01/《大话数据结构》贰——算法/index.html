算法定义

算法是解决特定问题求解步骤的描述,在计算机中表现为指令的有限序列,并且每条指令表示一个或多个操作.
不同的问题一般会有不同的算法,同一个问题有事也会有很多种算法,这时候就要具体情况具体分析,选出最合适的那个.

算法的特性

算法具有五个基本特性: 输入、输出、有穷性、确定性和可行性

  • 输入输出: 算法具有另个或多个输入，至少有一个或多个输出。

  • 有穷性：值算法在执行有限的步骤之后，自动结束而不会出现无限循环，并且每一个步骤课在接受的时间内完成。

  • 确定性：算法的每一步骤都具有确定的含义，不会出现二义性。

  • 可行性：算法的每一步都必须是可行的，也就是说，每一步都能通过执行有限次数完成。

算法设计的要求

算法不是唯一的，没有标准答案，但也有其需要满足的要求。

  • 正确性：算法的正确性是指算法至少应该具有输入、输出和加工处理无歧义性、能正确反映问题的需求、能够得到问题的正确答案。
  • 可读性： 算法设计的另一目的是为了便于阅读、理解和交流。
  • 健壮性： 当输入数据不合法事，算法也能做出相应的处理，而不是产生异常或莫名其妙的结果。
  • 时间效率高和存储量低

事后统计方法

事后统计方法： 这种方法主要是通过设计好的测试程序和数据，利用计算机计时器对不同算法编制的程序的运行时间进行比较，从而确定算法效率的高低。
缺陷：

  1. 需依据算法实现编好程序，不仅花费大量时间精力，且如果算法不满足需求，就白忙活一场了。
  2. 在不同的机器上，相同的算法的效率会有差异，有时会掩盖算法本身的优劣。
  3. 算法的测试数据设计困难，并且程序运行时间往往和测试数据的规模有很大关系，越高效的算法，越需要大规模的实验数据才能比较差异。

事前分析估计方法

事后分析估计方法： 在计算机程序便之前，依据统计方法对算法进行估算。
一个高级程序语言编写的程序在计算机上运行是所消耗的时间取决于下列因素：

  1. 算法采用的策略、方法。
  2. 编译产生的代码质量。
  3. 问题的输入规模。
  4. 及其执行指令的速度。

其中，第2条有编译软件来支持，第4条有硬件性能支持。所以，在分析估计算法质量时，我们主要看问题的输入规模。

最坏情况与平均情况

最坏情况运行时间是一种保证,在实际应用中是一种最重要的需求,通常除非特别指定,我们提到的运行时间都是最坏情况的运行时间.
平均时间是所有情况中做有意义的,因为他是期望的运行时间,但平均时间一般难以通过分析得到,需通过数据估算得出.
一般在没有特殊情况的说明下都是值最坏时间复杂度,不仅易于分析计算,又能满足甲方各种刁难.

算法空间复杂度

算法的空间复杂度通过计算算法所需的存储空间实现,算法空间复杂度的计算公式基座:S(n) = O(f(n)),其中,n为问题的规模,f(n)为语句关于n所占存储空间的函数.

总结回顾

  1. 数据结构与算法是相互依赖不可分割的
  2. 算法的定义: 算法是解决特定问题求解步骤的描述,在计算机中为指令的有限序列,并且每条指令表示一个或多个操作.
  3. 算法的特性: 有穷性、确定性、可行性、输入、输出
  4. 算法的设计的要求： 正确性、可读性、健壮性、高效率和地存储量需求。
  5. 算法特性与算法设计容易混淆，需对比记忆（好像我并不知道什么事算法设计）。
  6. 算法的度量方法： 时候统计方法（不科学、不准确）、事前分析估算方法。
  7. 函数的渐进增长： n>N时,f(n)>g(n),就说f(n)的增长渐进快于g(n).随着n值的增大,优的算法会越优,差的算法会越差.
  8. 算法时间复杂度T(n) = O(f(n)),推导大O阶的步骤
    • 用常数1取代运行时间中的所有甲方常数.
    • 在修改后的运行次数函数中,只保留最高阶项.
    • 如果最高阶项存在且不是1,则去除与这个项相乘的常数.
    • 常见的复杂度排序:
      $0(1) < O(logn) < O(n) < O(nlogn) < 0(n^2 ) < 0(n^3 ) < 0(2^n) < O(n!) < O(n^n)$
