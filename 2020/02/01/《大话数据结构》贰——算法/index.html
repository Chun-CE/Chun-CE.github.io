数据结构与算法关系

在学习数据结构是经常会涉及到一些算法以及他们之间的关系.

两种算法的比较

例: 现要求编写程序实现求出1+2+3+…+100的和
基础算法:

int i,sum = 0,n = 100;
sum = (1 + n) * n/2;
printf(“%d”,sum);

int i,sum = 0,n = 100; //执行 1 次
for (i = 1;i <= n;i++) //执行 n+1 次
{
sum = sum + i; //执行 n 次
}
printf(“%d”,sum); //执行 1 次

int sum = 0,n = 100; //执行 1 次
sum = (1 + n) * n/2; //执行 1 次
printf(“%d”,sum); //执行 1 次

```
显然,第一种算法共执行了 1+(n+1)+n+1 = 2n+3 次;而第二种算法共执行了 1+1+1 = 3 次;随着数据的增大,2n+3和3的差距会越来越大.

函数的渐进增长

现有两个算法A和B,算法A需执行 2n+3 次操作,算法B需执行 3n+1 次操作.
速度比较:

次数 算法A(2n+3) 算法B(3n+1)
n = 1 5 4
n = 2 7 7
n = 3 9 10
n = 10 23 31
n = 100 203 301

可以看出n<2时,B更快;随着n的增加,当n>2时A更快,而且与B的差距越来越大.
给出定义如下,输入规模n在没有限制的情况下,只要超过一个数值N,这个函数就总是大于顶一个函数,我们称这个函数是渐进增长的.
现令算法C是4n+8,算法D是 $2n^2$
